## Lecture Background
So we are living in a complicated era for developers.
For years we have learned to develop code, debug it, test it and then deploy it.
Today there is a new player in the game - AI.
and people say that he does it better and faster than us.
This is globally true but need to remember that AI is only a tool. A a tool need someone who knows hot to operate right to achive results.
Developing using artifical intelligence is considered to be a new approach toward software development. 
But surprisingly, An AI assisted software development have more in common with traditional software development than we think.
So how we approach this ?
Luckily, there is a lot of foundation that we can build on.

But first you should be understanding that you don't need to think as a software developer anymore.
Why ? because AI can do it for you. and it does it better.

So how we approach this ?
Your mindset should be that of a software engineer. 
An enginner doesn't write code. Never. He designs systems, architectures, and solutions.

Another mindset is a Project Manager. You need to understand the Software Development Life Cycle (SDLC).
If today we need to have a team leader and 4 developers to develop a complex application, with AI we can have 1 developer only proficient with AI assisted development.

As an engineer you should be familiar with technologies, frameworks and tools. 
You should be familiar with coding as well but you don't need to be a master in it.
You also need to develop debugging and problem-solving skills.

As a project manager you should be familiar with the SDLC, breaking down features into tasks, and Planning.
This is not something a developer today needs to do, but with AI assisted development your speed and capacity as so much higher that you must be able to manage larger projects.
You are not responsible for specific task but to a whole features and deliverables. and this scope should be managed more carefully.


Before we dive into practical example, let's see a brief overview of the SDLC.

Phase 1: Idea
Phase 2: Specification
Phase 3: Design
Phase 4: Implementation Planning
Phase 5: Implementation
Phase 6: Verification
Phase 7: Validation
Phase 8: Deployment
Phase 9: Maintenance

Now we do a practical example in the IDE.

Conclusion: 
- AI assisted software development is a reality
- It requires a different mindset from the one we currently have
- We need to think as engineers and project managers, not as developers
- We need to be able to break down features into tasks and manage them
- We need to be able to understand the traditional software development process to be able to use AI effectively
